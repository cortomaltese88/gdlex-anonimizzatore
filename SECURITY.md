# Security Policy

## Ambito

`gdlex-anonimizzatore` puo' trattare documenti che contengono dati personali, dati giudiziari o altre informazioni sensibili. Anche report e sessioni JSON possono includere valori originali o metadati delicati.

## Segnalazioni

- Non inserire dati reali, estratti cliente o contenuti riservati in issue, ticket pubblici o discussioni aperte.
- Non allegare documenti cliente, report reali o sessioni JSON provenienti da casi effettivi.
- Per vulnerabilita', problemi privacy o esposizioni di dati usare un canale riservato concordato con il manutentore del progetto.

## Uso prudente

- Verificare sempre manualmente i documenti anonimizzati prima di condividerli o depositarli.
- Trattare output, report e sessioni esportate come materiale potenzialmente sensibile.
- Ridurre al minimo i dati di test e preferire campioni sintetici o anonimizzati a monte.
