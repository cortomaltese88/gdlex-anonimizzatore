# Contributing

Contributi via pull request sono preferiti.

## Linee guida

- Non committare documenti reali, dati cliente, report reali o sessioni JSON provenienti da casi effettivi.
- Usare file sintetici o fixture di test minimali.
- Mantenere o aggiornare i test quando un cambiamento altera il comportamento osservabile.
- Evitare modifiche massive o refactor estesi se non motivati dal problema da risolvere.
- Aggiornare `README.md` e la documentazione in `docs/` quando cambiano flusso d'uso, prerequisiti o comportamento dell'app.

## Ambito preferito delle PR

- correzioni mirate;
- miglioramenti documentali;
- test aggiuntivi;
- affinamenti UI o UX circoscritti e verificabili.
